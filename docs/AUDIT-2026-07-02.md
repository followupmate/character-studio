# Repo Audit & Improvement Plan — Character Studio

**Dátum:** 2026-07-02 · **Vetva:** `claude/engine-improvement-wo6ivx` · **Režim:** len analýza, žiadne zmeny kódu

## Executive Summary

**Známka: C+.** Funkčne bohatý, rýchlo sa hýbuci single-operator produkt s nadpriemernou dokumentačnou kultúrou (HANDOFF/ROADMAP/inline komentáre vysvetľujúce *prečo*) a čistou doménovou vrstvou v `lib/` — ale beží ako **verejne prístupný systém s reálnymi peniazmi bez akejkoľvek autentifikácie, bez jediného testu a bez CI**. 39 zo 45 API routes je úplne otvorených, vrátane mazania charakterov, pálenia API kreditov a publikovania na živý Instagram a Fanvue účet. Zvyšných 6 routes má „auth", ktorý zlyháva do otvoreného stavu, a najdrahšia route sa dá obísť podvrhnutou Origin hlavičkou. **Top 3 riziká:** (1) otvorený API povrch = ktokoľvek s URL vie mazať dáta, míňať kredity a postovať na živé účty; (2) žiadna záchranná sieť — regresia v publish pipeline ide rovno do produkcie; (3) RLS vypnuté na 11 tabuľkách + verejný anon kľúč. **Top 3 príležitosti:** (1) jeden `middleware.ts` + zdieľaný auth helper uzavrie celý povrch za pol dňa; (2) jadro biznis logiky sú čisté funkcie — testy sú lacné a vysoko hodnotné; (3) konsolidácia 5× skopírovaných helperov výrazne zlacní každú ďalšiu zmenu.

## 1. Repo Map

**Účel:** Multi-character AI content platforma — generuje denné IG/YT batche pre AI influencerov (Claude príbehy + Higgsfield/Google/fal obrázky a videá), publikuje cez Graph API, monetizuje cez Fanvue funnel (PPV posty/správy). Single-operator interný nástroj s produkčným dopadom (živé účty, reálne peniaze). Maturita: **produkčný prototyp** — beží naživo, iteruje sa denne.

**Stack:** Next.js 15 (App Router, TS strict), React 19, Tailwind 3, motion; Supabase (Postgres + Storage, prefix `chs_`, service key); Vercel (hosting + 5 cronov); externé API: Anthropic, Higgsfield, Google Gemini/Veo, fal.ai, BFL, IG Graph, YouTube, Fanvue (OAuth).

**Tok dát:** cron 06:00 `story` → `lib/dailyBatch` (scene brief + 8 slot promptov, 3-wave, per-slot retry) → UI `/today` alebo cron → `generate-media` (Google→Higgsfield→fal fallback, 3-worker pool) → `from-batch` → `chs_posts` queue → cron `publish/cron` každých 15 min (cron-job.org) → IG/YT. Paralelne: `import-insights` (20:00) vracia metriky → growth score; Fanvue vetva: unlock drafty → `/fanvue` approve → generate set → OAuth publish.

**Kľúčové adresáre:** `app/api/characters/*` (generovanie, 25 routes) · `app/api/publish/*` (queue + IG/YT integrácie, 13 routes) · `app/api/fanvue/*` (monetizácia, 4 routes) · `lib/` (doménová logika — prompty, tiery, skóre, klienti) · `components/dashboard|today/` (hlavné UI) · `supabase/migration.sql` (append-only idempotentná schéma) · `docs/` (HANDOFF, SCENARIO, session logy).

**Prekvapenia:** (a) žiadny `middleware.ts`, žiadny login — celé UI aj API sú verejné; (b) 0 testov, 0 CI pri systéme, ktorý postuje na živé účty; (c) 4 mŕtve wizard stránky zo starého flow; (d) nezvyčajne dobrá dokumentácia rozhodnutí.

**Hĺbka review:** najhlbšie core (generate-media, dailyBatch, publish pipeline, fanvue vrstva, wizard, dashboard). Ľahší prelet: `generatePrompts.ts` (892 LOC), `slotPrompts.ts`, `ReviewClient`, `CharacterGrid`, `metrics/growth` UI.

## 3. Improvement Strategy

### Téma 1 — „Appka verí sieti" (S1–S4)
**Cieľový stav:** celý povrch fail-closed za jedným zdieľaným mechanizmom: `middleware.ts` s password cookie pre UI + `lib/auth.ts` (jediná implementácia, secret **povinný**) pre API/crony. **Princíp:** single-operator nástroj nepotrebuje user accounts — potrebuje jeden zámok na dverách a nulu vedľajších vchodov.

### Téma 2 — Peniaze bez záchrannej siete (T1–T2)
**Cieľový stav:** CI gate (typecheck + lint + build) na každom PR + unit testy pre 6 čistých money-modulov. **Princíp:** testovať tam, kde je pomer hodnota/úsilie extrémny — čisté funkcie, ktoré rozhodujú o cene, cielení a sanitizácii.

### Téma 3 — Copy-paste entropia (A2, A3)
**Cieľový stav:** jeden `lib/auth.ts`, jeden `lib/promptClean.ts`, jeden `lib/slots.ts`; mŕtve wizard stránky zmazané. **Princíp:** konsoliduj to, čo sa mení často; nechaj tak to, čo je len veľké.

### Téma 4 — Prostredie ako zdroj právd (O1, A6, S4)
**Cieľový stav:** kompletný `.env.local.example`, `lib/env.ts` s validáciou pri štarte, README sekcia „Env & Setup". **Princíp:** nové prostredie sa musí dať postaviť za 15 minút bez archeológie.

### Čo zámerne NEFIXOVAŤ teraz (trade-offs)
- **Big-bang refactor PublishClient/generate-media** — funguje, HANDOFF explicitne varuje pred redesignom publish pipeline; deliť až pri najbližšej väčšej zmene daného súboru (Boy Scout rule).
- **Next 16 / ESLint 9 major upgrade** — žiadny akútny prínos, netriviálne riziko; po M1.
- **User accounts / multi-tenant auth** — over-engineering pre single-operator; password gate stačí.
- **Observability stack (Sentry…)** — odložiť, kým cron zlyhania nezačnú reálne bolieť; lacnejšia medzistanica je error stĺpec + dashboard badge.
- **RLS zapnutie** — pripravené, čaká na explicitné rozhodnutie majiteľa (už 2× odložené).

### Definícia „hotovo" (merateľné signály)
1. `curl` na ľubovoľnú mutujúcu/drahú route bez secretu → **401** (dnes 200).
2. UI bez password cookie → redirect na login (dnes 200).
3. CI: PR bez zeleného `typecheck+lint+build` sa nedá zlúčiť.
4. `npm audit --omit=dev` → 0 high.
5. Coverage ≥ 80 % na `growthScore`, `fanvueStrategy`, `fanvueUnlock`, `promptClean`, `from-batch:scheduledIso`, `create:PLATFORM_MAP`.
6. `.env.local.example` obsahuje 100 % používaných premenných.
7. 0 Critical nálezov pri re-audite.

### Implementačné náčrty — top 3

**M1-1 `lib/auth.ts` (fail-closed):**
```ts
export function requireSecret(req: Request): Response | null {
  const secret = process.env.CRON_SECRET;
  if (!secret) return NextResponse.json({ error: "CRON_SECRET not configured" }, { status: 503 }); // fail CLOSED
  const auth = req.headers.get("authorization");
  const url = new URL(req.url);
  if (auth === `Bearer ${secret}` || url.searchParams.get("secret") === secret) return null;
  return NextResponse.json({ error: "Unauthorized" }, { status: 401 });
}
export function requireSession(req: NextRequest): Response | null { /* cookie check, viď M1-2 */ }
```
Postup: (1) pridať helper; (2) mechanicky nahradiť 6 lokálnych `isAuthorized`; (3) do zvyšných 39 routes vložiť `const deny = requireSessionOrSecret(req); if (deny) return deny;` na prvý riadok handlera; (4) UI fetches netreba meniť — cookie ide automaticky. **Gotchas:** `publish/cron` volá cron-job.org s query `?secret=` — zachovať; OAuth callbacky (`/api/auth/*`) musia ostať bez session (stavajú ju… nie — sú chránené state/PKCE, nechať otvorené); po deployi okamžite overiť oba externé crony.

**M1-2 `middleware.ts`:** matcher na všetko okrem `_next/*`, `/login`, `/api/auth/*`; check httpOnly cookie `cs_session` (HMAC z `APP_PASSWORD`); API požiadavky bez cookie → skúsiť cron secret; inak 401/redirect. Login stránka = jeden input + server action, ktorá nastaví cookie. **Gotcha:** Vercel cron posiela `Authorization: Bearer CRON_SECRET` automaticky, len ak je env `CRON_SECRET` nastavené — čo M1-1 aj tak vynucuje.

**M0-1 CI:** `on: [push, pull_request]` → `npm ci`, `npx tsc --noEmit`, `npm run lint`, `npm run build` (env: dummy `SUPABASE_URL` atď. — build nesmie vyžadovať reálne secrety; ak vyžaduje, doplniť fallbacky do `lib/env.ts` z M2-5). **Gotcha:** `next build` prerendruje statické stránky — stránky čítajúce Supabase sú `force-dynamic`, takže build prejde aj s dummy env (overené: build beží lokálne bez env).
