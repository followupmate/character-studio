# Session 2026-07-02 — Engine → Money Maker

Súhrn všetkých zmien z tejto session (vetva `claude/engine-improvement-wo6ivx`),
setup kroky a čo treba spraviť ďalej.

---

## Diagnóza na začiatku

Engine vedel **vyrobiť** (charakter, denný 8-slot batch, prompty) a **publikovať**
(IG/YT crony), ale bol **otvorená slučka a konverzne slepý**:

1. Metriky sa nevracali — growth vrstva závisela od manuálneho importu, ktorý nikto nerobil → skóre večne 0, učenie sa nikdy neaktivovalo.
2. Fanvue (miesto, kde vznikajú peniaze) sa kŕmilo úplne ručne — drafty boli len text, publish bol zakázaný v kóde.
3. Most IG → Fanvue neexistoval — žiadne CTA v captionoch, story link sticker sa vypĺňal ručne z localStorage.
4. Peniaze neboli nikde vidieť.

Všetky štyri diery sú zaplátané.

## ČO TREBA SPRAVIŤ ĎALEJ (tvoje kroky, v poradí)

### A. Nasadenie
1. **Zlúč vetvu do main** — commity `a831afe`…`cdadea6` ešte nie sú v maine (PR #85–87 pokryli len prvé tri). Nový PR: https://github.com/followupmate/character-studio/pull/new/claude/engine-improvement-wo6ivx
2. Vercel nasadí automaticky; cron `import-insights` (20:00 UTC) sa aktivuje sám z `vercel.json`.

### B. Fanvue prepojenie (raz, ~10 minút)
1. Fanvue → **Builder area → Create app** (potrebuješ creator účet s hotovým KYC).
2. Redirect URI nastav na: `https://<tvoja-appka>.vercel.app/api/auth/fanvue/callback`
3. Client ID + Secret → Vercel env: `FANVUE_CLIENT_ID`, `FANVUE_CLIENT_SECRET` (voliteľne `FANVUE_SCOPES`, default: `offline_access read:self write:post write:media write:chat-message`).
4. Otvor `https://<tvoja-appka>.vercel.app/api/auth/fanvue` → povoľ prístup. Tokeny sa uložia do DB samé.
5. Over banner na `/fanvue` — má svietiť „Fanvue API pripojené (OAuth)“.

### C. Funnel aktivácia
1. Na `/fanvue` vyplň **IG → Fanvue link** pre charakter (Fanvue profil / tracking link) a ulož.
2. Ten istý link vlož **manuálne do IG bio** (jednorazovo).
3. Zapni flag, nech denný batch vyrába unlock drafty:
   ```sql
   UPDATE chs_characters
   SET feature_flags = coalesce(feature_flags, '{}'::jsonb) || '{"fanvue_drafts": true}'::jsonb
   WHERE slug = 'vivienne';
   ```
4. Voliteľne zapni aj `growth_layer` (tier bias sa aj tak aktivuje až po 10 skórovaných reeloch).

### D. Prvý ostrý beh (over a nahlás)
1. Po prvom publikovanom poste over o deň neskôr `/growth` a Money Engine panel — metriky majú pritiecť samé. Manuálny test: `GET /api/publish/import-insights?secret=<CRON_SECRET>&days=7`.
2. Prvý Fanvue publish: draft na `/fanvue` → „Vygeneruj set“ → „→ Fanvue post“. **Pozor:** presné cesty upload flow Fanvue nezverejňuje mimo docs portálu — klient skúša krátky zoznam kandidátov. Ak upload spadne, chyba sa zobrazí v karte draftu (`publish_error`) — **pošli mi ju, oprava je na 1 riadok** (`lib/fanvue.ts`, path konštanty).

### E. Ďalšie logické kroky (návrhy, zoradené podľa páky)
1. **Fanvue earnings auto-refresh** — denný cron cez OAuth API namiesto manuálneho MCP auditu → Money Engine panel vždy živý.
2. **PPV kadencia** — plánovač 2 unlocky/týždeň automaticky pripravené (set vygenerovaný) na approve.
3. **Welcome flow** — privítacia správa novým subscriberom (approve-gated).
4. **Fanvue tracking links** — kliky z IG do `engagement.fanvue_clicks` (váha 20× v growth skóre) → engine optimalizuje na konverziu, nie views.
5. **Video sloty** — ROADMAP bod 2 (dočasne vypnúť) je stále otvorené rozhodnutie majiteľa.
